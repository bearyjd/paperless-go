## Task 1: Verify Dart SDK compatibility with Flutter 3.41.3

**Files:**
- Read: `pubspec.yaml` (environment constraint is `sdk: ^3.9.2`)
- Possibly modify: `metadata/com.ventoux.paperlessgo.yml` (srclib version)

The F-Droid build uses `$$flutter$$` pinned at `flutter@3.41.3`. The pubspec requires Dart `^3.9.2`. If Flutter 3.41.3 bundles an older Dart, `flutter pub get` will fail with a Dart SDK version error — before any user code is even compiled.

- [ ] **Step 1: Check which Dart version Flutter 3.41.3 ships**

Run on a machine that has Flutter 3.41.3:
```bash
flutter --version
# Look for "Dart SDK version: X.Y.Z" — must be ≥ 3.9.2
```

Expected output (verify Dart ≥ 3.9.2):
```
Flutter 3.41.3 • channel stable
Dart SDK version: 3.9.x  ← must be ≥ 3.9.2
```

- [ ] **Step 2A (if Dart ≥ 3.9.2): no srclib change needed**

Proceed to Task 2.

- [ ] **Step 2B (if Dart < 3.9.2): bump the srclib version**

Find the earliest Flutter stable release that bundles Dart ≥ 3.9.2.  
F-Droid's supported Flutter srclib versions: check `https://gitlab.com/fdroid/fdroidserver/-/blob/master/srclibs/flutter.yml`

Update `metadata/com.ventoux.paperlessgo.yml`, changing the srclib in **both** the existing `v1.0.3` build entry **and** the new `v1.1.0` entry (Task 2) to the correct version:

```yaml
    srclibs:
      - flutter@3.XX.Y   # ← version that bundles Dart ≥ 3.9.2
```

- [ ] **Step 3: Commit if srclib was changed**

```bash
git add metadata/com.ventoux.paperlessgo.yml
git commit -m "chore: bump Flutter srclib to version that satisfies Dart ^3.9.2"
```

## Task 2: Add v1.1.0 build entry to F-Droid YAML

**Files:**
- Modify: `metadata/com.ventoux.paperlessgo.yml`

This is the critical missing piece. Without a `Builds:` entry for versionCode 5, F-Droid will see `CurrentVersion: '1.1.0'` but have no recipe to build it — the submission MR will be blocked.

The prebuild script is identical to versionCode 4: strip `google_mlkit_text_recognition` from pubspec.yaml, then copy both stub files over the ML Kit implementations.

- [ ] **Step 1: Open `metadata/com.ventoux.paperlessgo.yml` and add the v1.1.0 build entry**

Insert the new entry **after** the existing `v1.0.3` entry, immediately before the `AutoUpdateMode:` line:

```yaml
  - versionName: '1.1.0'
    versionCode: 5
    commit: v1.1.0
    output: build/app/outputs/flutter-apk/app-release.apk
    srclibs:
      - flutter@3.41.3
    rm:
      - ios
      - linux
      - macos
      - web
      - windows
    prebuild:
      - export PUB_CACHE=$(pwd)/.pub-cache
      - $$flutter$$/bin/flutter config --no-analytics
      # Remove proprietary Google ML Kit dependency for F-Droid FOSS compliance.
      # Replace ML Kit files with pure-Dart stubs that gracefully degrade:
      # - Deskew falls back to projection-profile algorithm (pure Dart)
      # - OCR metadata suggestions are disabled (returns empty text)
      - sed -i '/google_mlkit_text_recognition/d' pubspec.yaml
      - cp lib/features/scanner/processing/mlkit_deskew_stub.dart lib/features/scanner/processing/mlkit_deskew.dart
      - cp lib/features/scanner/processing/ocr_extractor_stub.dart lib/features/scanner/processing/ocr_extractor.dart
      - $$flutter$$/bin/flutter pub get
    scandelete:
      - .pub-cache
    build:
      - export PUB_CACHE=$(pwd)/.pub-cache
      - $$flutter$$/bin/flutter build apk --release
```

This simulates what F-Droid's CI will do, minus the Flutter srclib injection. Run from the repo root:

```bash
# Work in a temp copy so we don't destroy the working tree
cp -r . /tmp/paperless-go-fdroid-test
cd /tmp/paperless-go-fdroid-test

# Step 1: strip ML Kit from pubspec
sed -i '/google_mlkit_text_recognition/d' pubspec.yaml

# Verify it's gone
grep google_mlkit pubspec.yaml && echo "FAIL: still present" || echo "OK: removed"

# Step 2: copy stubs over implementations
cp lib/features/scanner/processing/mlkit_deskew_stub.dart \
   lib/features/scanner/processing/mlkit_deskew.dart

cp lib/features/scanner/processing/ocr_extractor_stub.dart \
   lib/features/scanner/processing/ocr_extractor.dart

# Step 3: verify stub content is active (should show "return null" / "return ''")
head -5 lib/features/scanner/processing/mlkit_deskew.dart

# Step 4: pub get with local flutter
flutter pub get

# Expected: resolves without errors. If it fails with "no matching version"

cd -
rm -rf /tmp/paperless-go-fdroid-test
```

- **"Could not find package google_mlkit_text_recognition"** → `sed` missed a line; check for duplicates in pubspec.yaml
